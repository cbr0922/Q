How to install
If you have purchased the bundle first set up nuevoMailer and then return here and continue reading.
Upload only these files in the admin folder of your nuevoMailer installation. Overwrite the old ones if asked.
_schedulerCreateTaskExec.php
_schedulerCreateTaskForm.php
_schedulerEditTaskExec.php
_schedulerEditTaskForm.php
_schedulerSend.php
_schedulerStart.php
_schedulerTaskNotes.php
_schedulerTasks.php
You must give "modify" and "write" permissions to the data_files folder.
Then login and go Menu>Tools>Campaign scheduler.
First you create a campaign and then you schedule it by creating a task for this campaign.
Notes
Batch settings: the global batch settings do not apply when using the scheduler. But you can define batch settings specific to a task.
The pause between batches (batch interval) is related to the cron timer in this manner: if you set a batch interval of 10 minutes but the cron hits every 30 minutes it is evident that sending will start every 30 minutes. So the cron timer should be set to a value close to the batch pause. Lower is fine.
SMTP anti-flood settings will be used if defined for an smtp server.
Cron set-up
Two alternatives are provided.
Alternative 1 (easier and recommended)
Alternative 1 is simpler and easier to set up. Use it when you are not limited by the number of cron jobs that you can set up using your Host's control panel. You create one cron job only for the purpose of campaign scheduling.
This is the command to run:
GET http://www.domain.com/nuevoMailer/admin/_schedulerStart.php?admin=yourUsername_yourPassword
Replace in the line above with your own domain, folder name, administrator username and password.
The username and password must be separated with an underscore: _
The GET command: if the alias command GET is not recognized by your server replace GET with curl -L -s or WGET.
Alternative 2
With the second alternative you create one cron job but the file you hit can be used for more than one tasks.
Example: it can also be used for bounce processing when using the Bounce Manager.
Open the file taskChecker.php. You will see this line:
GET http://www.domain.com/nuevoMailer/admin/_schedulerStart.php?admin=yourUsername_yourPassword
You may also see this line used for bounce processing (and this is the main benefit of alternative 2):
GET http://www.domain.com/nuevoMailer/admin/_bmProcessCron.php?admin=yourUsername_yourPassword
Replace in the line above with your own domain, folder name, administrator username and password.
The username and password must be separated with an underscore: _
For security, the file taskChecker.php must be placed outside public_html or anywhere not accessible to others.
Permission for taskChecker.php should be 755
The Cron command is:
/home/username/etc/taskChecker.php
Adjust the path accordingly and replace in the above the username and folder name: /username/etc/
Examples:
/home/jim/etc/taskChecker.php
/home/jim/cron/taskChecker.php
Cron timing
Depends on your needs.
if you have several mailings scheduled throughout the day then you can set it to run every 10 minutes or so. It is not resource intensive. First it checks for tasks. if it finds any it process them, otherwise it stops. The log file provides extensive details about each step.
Set up via command line
If you do not have a friendly interface to enter the command and timing then you can refer here for details or contact us. http://en.wikipedia.org/wiki/Crontab.
You may also use this free service to create your cron command http://www.htmlbasix.com/crontab.shtml.
Troubleshooting
The first place to look is the scheduler log file. You will find a direct link to this file at the scheduler tasks page in your administration panel.
If you don't see (or cannot find) this file then most likely the data_files folder does not have write permissions.
To verify this, copy the whole cron command, paste it in your browser's address bar and hit it. You will get a descriptive message.
