How to install
If you have purchased the bundle first set up nuevoMailer and then return here and continue reading.
Upload only these files in the admin folder of your nuevoMailer installation. Overwrite the old ones if asked.
_bm.php
_bmProcess.php
_bmProcessAdmin.php
_bmProcessCron.php
_bmRemove.php
_bmSigs.xml
How to use
Before sending a campaign go to Menu>Tools>Configuration & settings and define the email account were bounces will be sent. At the bottom of the same page complete all the required credentials so that bounce manager can login and read this email account.
These credentials are similar to what you would use to set-up this email account in your email program.
The bounce manager reads this mailbox.
To test your settings set the "Number of messages to process in a single run" to 0.
Then go Menu>Tools>Process bounced emails and click on "Click here to process bounced emails".
You will understand from the message that you will see if it works or not.
In case of a problem double check your settings and also experiment with port numbers 110 or 143.
The bounce manager only flags the subscribers with soft or hard bounces but it does not automatically delete them. You must do that manually using the available functions in nuevoMailer. Or instead of deleting them you may want to set them as un-verified or add them to the suppression list.
Using filters you can view these subscribers before deleting them.
If you want the bounce manager for an earlier version of nuevoMailer then contact DesignerFreeSolutions.
Bounce processing with Cron
This is optional and may come handy. You do not have to set it up. You can always do manual processing from your admin panel. It will be very useful for processing large volumes of bounces. Two alternatives are provided.
Alternative 1 (easier and recommended)
Alternative 1 is simpler and easier to set up. Use it when you are not limited by the number of cron jobs that you can set up using your Host's control panel. You create one cron job only for the purpose of bounce processing. How often this job should run depends on the volume and frequency of your campaigns.
This is the command to run:
GET http://www.domain.com/nuevoMailer/admin/_bmProcessCron.php?admin=yourUsername_yourPassword
Replace in the line above with your own domain, folder name, administrator username and password.
The username and password must be separated with an underscore: _
The GET command: if the alias command GET is not recognized by your server replace GET with curl -L -s or WGET.
Alternative 2
With the second alternative you create one cron job but the file you hit can be used for more than one tasks.
Example: it can also be used for scheduling when using the Campaign Scheduler.
Open the file taskChecker.php. You will see this line:
GET http://www.domain.com/nuevoMailer/admin/_bmProcessCron.php?admin=yourUsername_yourPassword
You may also see this line used for scheduling (and this is the main benefit of alternative 2):
GET http://www.domain.com/nuevoMailer/admin/_schedulerStart.php?admin=yourUsername_yourPassword
Replace in the line above with your own domain, folder name, administrator username and password.
The username and password must be separated with an underscore: _
For security, the file taskChecker.php must be placed outside public_html or anywhere not accessible to others.
Permission for taskChecker.php should be 755
The Cron command is:
/home/username/etc/taskChecker.php
Adjust the path accordingly and replace in the above the username and folder name: /username/etc/
Examples:
/home/jim/etc/taskChecker.php
/home/jim/cron/taskChecker.php
Cron timing
Depends on how often and how big your mailings are.
Since bounces do not always return instantly you can have it run every 30 minutes or more. It is better to activate it after a mailing has finished. For large volumes of bounced emails it is advised to set a low pop3 batch size (in your configuration settings) and a shorter time interval (for the CRON period).
Set up via command line
If you do not have a friendly interface to enter the command and timing then you can refer here for details or contact us. http://en.wikipedia.org/wiki/Crontab.
